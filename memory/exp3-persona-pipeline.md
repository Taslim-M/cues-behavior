**Experiment 3** = repeat of Experiment 2 on the same `generate_synthetic_data/prompts.json` factorial, but the self-modeling step uses `infer_persona_prompt` (model names a persona + 4–6 load-bearing traits) instead of the contextual-cue prompt. Judge = `anthropic/claude-sonnet-4.5` scoring warmth/formality/advice_density/primary_emotion (0–10) on the response only.

**Data change (2026-07-09):** scenarios `savings_milestone`, `sister_age`, `students_passed` were REMOVED from `prompts.json` (3240→2448 rows; now **9 scenarios**). `select_balanced` now yields 12 roles × 3 eval_conditions × 9 scenarios = **324 prompts**. The 3 scenarios were also pruned from the old `results/exp3_persona/` jsonl aggregates + per-datapoint json (moved to a scratchpad backup); the old stage2/cf/xsweep summary JSONs + `analysis_persona_synth.html` are now STALE and would need a rebuild to match.

**DECOUPLED design (2026-07-09, new):** the coupled design (persona + reply in one generation) can let writing the persona steer/rationalise the reply. The decoupled runner separates them into 3 independent calls per prompt: persona-ONLY elicitation with `persona_v1` (staging framing) AND `persona_v3` (self-report framing) — trait 0–10 values AVERAGED across the two (`merge_traits`, union by normalized name, `n_sources`=2 when both name it) — plus a natural response with NO self-modeling scaffold (only the eval_condition frame kept, so the factorial is preserved). Judge scores the natural reply.
- New files: `src/run_exp3_decoupled.py`; `system_prompts.py` → `PERSONA_SYSTEM_PROMPT_V1/_V3` (keys `persona_v1`/`persona_v3`); `config.EXP3_DECOUPLED_DIR = results/exp3_decoupled`.

**Code (all under `src/`):**
- `system_prompts.py` → `PERSONA_SYSTEM_PROMPT` (key `"persona"`)
- `inference.py` → `parse_persona()` (extracts `<persona>`/`<response>`, groups `trait_N_*`)
- `run_exp3_persona.py` → Step 1 (generate+parse+judge)
- `analyze_exp3_persona.py` → Step 2 (persona inventory, ANOVA η², persona-text→DV ridge, trait-value→DV corr) → `results/exp3_persona/stage2_summary.json` + `figures/`
- `persona_cf.py` + `run_exp3_persona_cf.py` → Step 3 counterfactuals (trait_suppress / persona_swap / null_persona) → `results/exp3_persona/cf_summary.json` + `cf/`
- `run_exp3_xsweep.py` + `analyze_xsweep.py` → dose-response: hold eval_condition=deployment, sweep ALL x-values per scenario (936 prompts/model) → `results/exp3_persona/xsweep/` + `xsweep_summary.json` + figures
- `make_persona_report.py` → builds interactive `analysis_persona_synth.html` at repo root (tabs: overview/method/examples/personas/drivers/cross-model/faithfulness/counterfactuals/dose-response/figures)

**Run:** foreground, per model (see [[openrouter-runs-foreground]]). Ran runs=1 (full factorial) per model + cap-8 counterfactuals. All 4 models: 432/432 Step-1, 0 failures.

**Headline findings (2026-06-25):** models collapse into a Cautious-Advisor vs Empathetic-Supporter split; scenario(topic) dominates DVs, eval-frame ≈ 0 effect; verbalized persona is correlationally faithful (CV R²≈0.7–0.8). Counterfactually, **persona_swap** is a strong faithful lever (advice Δ −5.7 llama-3.3-70b, −7.8 qwen3-30b, faithfulness 1.00); **single-trait suppression barely moves the gestalt**. Steerability varies: qwen3-30b high (0.84), qwen3-235b largely ignores edited persona (0.33). **Dose-response (x-sweep):** persona is set by topic not magnitude — pooled severity→DV ≈0, but split by valence it's sensible (pos scenarios: advice falls as outcome improves, e.g. students_passed r=−0.43; drink_drive r=+0.31; dog_missing r=−0.37); the character almost never flips as stakes rise. **llama-3.1-8b barely emits a `<persona>` block (≈99% empty in deployment)** — it ignores the persona self-modeling format, so its persona-level analyses are null (DVs still fine).
